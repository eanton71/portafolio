- Javascript: lenguaje de script 
- Creado para dar interactividad a las pagina web 
- Los programas se escriben en trozos de codigo llamados scripts. 
- Se pueden intergrar en el codigo html o en archivos externos ".js"
- Se ejecutan en texto plano. No son compilados. Son interpretados
## Historia
- Javascript no tiene que ver nada con Java (o muy poco) 
- Fue creado  en 1995 por Brendan Eich, con el nombre de "LiveScript" y se integro en el uso del navegador Netscape Navigator
- Microsoft crea posteriormente JScript para Intenet Explorer 3.0 
- En 1997 Ecma international publica el estandar ECMA-262 del lenguaje ECMAScript. El objetivo era tener un estandar comun para poder usar en todos los navegadores.
- Surgen dialectos basados en este estandar, aparte de JScript, como ActionSCript de Adobe.
- Se suceden diferentes versiones: 
    - ES5 del 2009
    - ES6 o ES2015, 2015
    - Despues ha habido versiones anuales: ES2017, ES2018, ...

## Como funciona
- El codigo es leido por el navegador junto con el resto delo contenido de la pagina web: html, css, imagenes,... 
- Se lee cmo una secuencia de caracteres Unicode. 
- El interprete, un vez recibe una secuencioa de comandos,    realiza un analisis lexico y oconvierte la cadena a: 
    - tokens
    - caracteres d control de formato
    - terminadores de linea- comentarios
    - espacios en blanco
## Sintaxis 
- Declaraciones 
    - Instruciciones en una o varias  lineas de cocidgo que representan una accion: asignaiones de variables, 
    - Las instruccioes deben terminar en punto y coma (no es obligatorio pero si recomendable)
- Sentencias de bloque: condicionales, bucles, funciones. 
- Expresi칩n: unidad de codigo que genera un valor. Los parentess `()`  se usan para agrupar partes de una expresion para  que esta s eejecute de una vez, para hacerla mas legible o para sobreescribir el orden de operadores matematicos. 
- Tipado debil: no es necesario declarar los datos con un tipo especifico. El interprete puede deducir el tipo a partir del contedo y su contexto (coerci칩n de tipos) `"1" + 1 > `<samp>"11"</samp>.
- Los tipos tambien se pueden convertir explicitamente mediante metoddos como toString(). 
- Sensible al uso de mayusculas y minusculas (case sensitivity). No es lo mismo `console.log`que `console.Log` -> <samp>Error</samp>
- Los espacios no son interpretados. 
```js
console.log(     "hola"   );console.log(   "adios"   );
>"hola"
>"adios"
```
- Si lo son cuando separan tokens  `let x`. Deben llevar al menos un espacio entre ellos. Tambien pueden llevar saltos de linea
- El interprete de Javascript se encarga e eliminar los espaios innecesarios. Aunque esto es mas legible. 
```js

  //some behavior;
```


- El uso de sangria y tabuladores es recomendable para mejorar la legibilidad del codigo aqunque el interprete no los necesite. 

## Como se ejecuta 
- El interprete de Jacvscript es llamado **motor o interprete de Javascript**.
- Esta  integrado por lo general en navegadores, pero tambien puede estarlo en servidores o en algunos dispositivos.
- Los motores que llevan los navegaddores tienen difrentes nombres:
    - **V8**: Chrome, Opera, Edge.
    - **SpiderMonkey**: Firefox. 
    - **Chakra**: IE, **SquirrelFish** en Safari,...
- Funcionamiento:
    - El motor analiza el script.
    - Lo compila a codigo maquina.
    - Se ejecuta este codigo. 
## Que puede hace Javascript 
- Fue creado para su uso en navegadores por tanto no reuere acceso a recursos como memoria, CPU,...
- Segun el entorno de ejecucion tiene distitnatas capacidades. Node.JS permite implementar un servidor y tiene acceso a archivos, realiazar solicitudes de red,..
- En el navegador esta limitado a la manipulacion de la pagina web en la que esta integrada, interacciones con el usuario y el servidor web. 
- Puede configurar cookies  que son unbos archivos limitados de configuracion o guardar datos en **local storage** limitados a una pagina web en concreto. 
- No puede acceder al disco duro o a funciones del sistema operativo. uede trabajar con archivos de forma limitada y con permiso expreso del usuario . Acciones integradas en ciertas etiquetas html como `<input>`.
- Se requiere permiso para hacer uso de camara, microfono y dispositivos que puedan ser de uso de la pagina web. Esta por si misma no puede activarlos. 
- Desde un script de una pagina no se puede aceder  a la pesta침a que este abierta en el navegador por parte de otra pagina . 
- Esto se llama politica del mismo origen, ("Same Origin Policy"). Se pueden comunicar diferentes dominios pero mediante un protocolo especifico. 
- Si no fuera asi, una pagina abierta en una pesta침a podria acceder a otra que tuviera informacion personal de l usuario como el gestor de correo electronico.
